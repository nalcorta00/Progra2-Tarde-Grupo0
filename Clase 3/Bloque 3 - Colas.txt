Fila del Cajero:
Se implementa una estructura de cola convencional basada en el principio FIFO (First-In, First-Out). En este modelo, el orden de atención se rige estrictamente por el orden de llegada: el primer usuario en ingresar es el primero en ser atendido, sin que existan prioridades jerárquicas para otros individuos.

package Implementacion;

import Interfaces.ColaTDA;

public class FilaCajero implements ColaTDA {

    private static final int MAX = 50; // Capacidad máxima de la fila
    private int[] alumnos;
    private int cantidad;

    @Override
    public void InicializarCola() {
        alumnos = new int[MAX];
        cantidad = 0;
    }

    @Override
    public void Encolar(int idAlumno) {
        if (cantidad < MAX) {
            // El alumno nuevo se ubica al final de la fila actual
            alumnos[cantidad] = idAlumno;
            cantidad++;
            System.out.println("Alumno " + idAlumno + " ingresó a la fila.");
        } else {
            System.out.println("Fila llena: el alumno debe esperar afuera.");
        }
    }

    @Override
    public void Desencolar() {
        if (!ColaVacia()) {
            System.out.println("Atendiendo al alumno en el cajero...");
            // Al salir el primero (índice 0), todos los demás avanzan un lugar
            for (int i = 0; i < cantidad - 1; i++) {
                alumnos[i] = alumnos[i + 1];
            }
            cantidad--;
        }
    }

    @Override
    public int Primero() {
        // Devuelve el alumno que está al frente (el próximo a ser atendido)
        return alumnos[0];
    }

    @Override
    public boolean ColaVacia() {
        return cantidad == 0;
    }
}

En este escenario, se implementa una cola de impresión convencional sin jerarquías. El sistema procesa los documentos siguiendo un orden estrictamente secuencial: el primer archivo en ser encolado será el primero en imprimirse, sin considerar criterios de prioridad adicionales.

Guardia De Hospital: 
Se implementa una cola con prioridad donde el orden de atención es determinado por la urgencia del cuadro clínico. A diferencia de una cola simple, aquí los elementos se reordenan dinámicamente según su valor de prioridad:

Prioridad 100 (Crítica): Paciente con infarto agudo de miocardio. Representa un riesgo de vida inmediato.

Prioridad 90 (Alta): Embarazo en fase de parto inminente. Se prioriza el bienestar de dos individuos y la urgencia del procedimiento, aunque el riesgo vital suele ser menor que en un paro cardíaco.

Prioridad 1 (Baja): Paciente con herida leve (raspón). No presenta riesgo sistémico y puede esperar a que los casos críticos sean resueltos.

package Implementacion;

import Interfaces.ColaPrioridadTDA; // Asumiendo que tenés la interfaz

public class GuardiaHospital implements ColaPrioridadTDA {

    private static final int MAX = 100;
    private int[] valores;      // El ID del paciente o dato
    private int[] prioridades;  // La prioridad asociada
    private int cantidad;

    @Override
    public void InicializarCola() {
        valores = new int[MAX];
        prioridades = new int[MAX];
        cantidad = 0;
    }

    @Override
    public void EncolarPrioridad(int dato, int prioridad) {
        if (cantidad < MAX) {
            int i = cantidad;
            
            // Buscamos el lugar: movemos a la derecha mientras la 
            // prioridad nueva sea mayor a la de los que ya están.
            // Esto deja al de MAYOR prioridad siempre en el índice 0.
            while (i > 0 && prioridades[i - 1] < prioridad) {
                prioridades[i] = prioridades[i - 1];
                valores[i] = valores[i - 1];
                i--;
            }
            
            valores[i] = dato;
            prioridades[i] = prioridad;
            cantidad++;
        }
    }

    @Override
    public void Desencolar() {
        if (!ColaVacia()) {
            // Como el más importante ya está en 0, 
            // solo corremos el resto a la izquierda (Estrategia 2).
            for (int i = 0; i < cantidad - 1; i++) {
                valores[i] = valores[i + 1];
                prioridades[i] = prioridades[i + 1];
            }
            cantidad--;
        }
    }

    @Override
    public int Primero() {
        // Gracias al ordenamiento al insertar, el de mayor prioridad es datos[0]
        return valores[0];
    }

    @Override
    public int Prioridad() {
        return prioridades[0];
    }

    @Override
    public boolean ColaVacia() {
        return cantidad == 0;
    }
}

Examen Final
Se presenta una cola con prioridad donde los alumnos promocionados tienen precedencia sobre aquellos en condición de examen final. No obstante, se aplica una regla de puntualidad: si un alumno promocionado llega con 30 minutos de demora, pierde su beneficio jerárquico y es desencolado después de los estudiantes que se presentaron puntualmente, independientemente de su condición.

package Implementacion;

import Interfaces.ColaPrioridadTDA;

public class GestionExamen implements ColaPrioridadTDA {

    private static final int MAX = 100;
    private int[] alumnos;      // Almacena el ID del alumno
    private int[] prioridades;  // Almacena 10 (Promocionado) o 5 (Regular)
    private int cantidad;

    @Override
    public void InicializarCola() {
        alumnos = new int[MAX];
        prioridades = new int[MAX];
        cantidad = 0;
    }

    @Override
    public void EncolarPrioridad(int idAlumno, int prioridad) {
        if (cantidad < MAX) {
            int j = cantidad;

            // Lógica de Desplazamiento:
            // Mientras el que está a la izquierda tenga MENOR prioridad que el nuevo,
            // lo movemos a la derecha para dejarle el lugar al más importante.
            while (j > 0 && prioridades[j - 1] < prioridad) {
                prioridades[j] = prioridades[j - 1];
                alumnos[j] = alumnos[j - 1];
                j--;
            }

            // El alumno promocionado (10) "salta" al regular (5) y queda más cerca del índice 0
            alumnos[j] = idAlumno;
            prioridades[j] = prioridad;
            cantidad++;
        }
    }

    @Override
    public void Desencolar() {
        if (!ColaVacia()) {
            // Estrategia 2: Al sacar al primero (indice 0), 
            // todos los demás avanzan un casillero a la izquierda.
            for (int i = 0; i < cantidad - 1; i++) {
                alumnos[i] = alumnos[i + 1];
                prioridades[i] = prioridades[i + 1];
            }
            cantidad--;
        }
    }

    @Override
    public int Primero() {
        return alumnos[0]; // Siempre devuelve el que tiene mayor prioridad
    }

    @Override
    public int Prioridad() {
        return prioridades[0];
    }

    @Override
    public boolean ColaVacia() {
        return cantidad == 0;
    }
}

Buffet Facultad
Para asegurar que nadie se cuele, se utiliza la operación encolar (enqueue). Debido a la naturaleza del TDA Cola, el acceso está restringido: los nuevos elementos solo pueden ingresar por el final de la estructura, respetando estrictamente el orden de llegada (FIFO).

Procesador De Tareas
Se implementa una cola con prioridad para la gestión de recursos del sistema. Bajo este modelo, los procesos críticos del sistema operativo poseen una jerarquía superior frente a aplicaciones de usuario (como Spotify). Ante una saturación de la estructura, el despacho de tareas se realiza priorizando el nivel de importancia y, en caso de empate en la prioridad, se respeta el orden de llegada (FIFO).

Embarque De Aerolineas
En este escenario se implementa una cola con prioridad estable. Ante la llegada de dos pasajeros con el mismo nivel de jerarquía (por ejemplo, dos pasajeros de clase "Business"), el sistema resuelve el conflicto mediante el orden cronológico de arribo. De esta manera, el primer pasajero en haberse encolado será el primero en ser procesado, garantizando una atención equitativa dentro de la misma categoría.

Distribucion De tickets IT
Se implementa un sistema de gestión basado en una cola con prioridad con un amplio rango jerárquico para clasificar la criticidad de los incidentes:

Prioridad 999 (Urgencia Crítica): Asignada al fallo del servidor principal. Al representar una interrupción total del servicio, este ticket se posiciona al frente de la cola para su resolución inmediata (Despacho prioritario).

Prioridad 0 (Nivel Cosmético/Bajo): Asignada a solicitudes estéticas como el cambio de fondo de pantalla. Al no afectar la operatividad, este elemento permanece en la estructura y solo es procesado cuando la cola de prioridades superiores se encuentra vacía.

package Implementacion;

import Interfaces.ColaPrioridadTDA;

public class GestionTicketsIT implements ColaPrioridadTDA {

    private static final int MAX = 500; // Capacidad del sistema de tickets
    private int[] idsTickets;
    private int[] nivelesPrioridad;
    private int cantidad;

    // Definición explícita de valores según la consigna
    private static final int PRIORIDAD_CRITICA = 999; // Servidor roto
    private static final int PRIORIDAD_BAJA = 0;      // Fondo de pantalla

    @Override
    public void InicializarCola() {
        idsTickets = new int[MAX];
        nivelesPrioridad = new int[MAX];
        cantidad = 0;
    }

    @Override
    public void EncolarPrioridad(int idTicket, int prioridad) {
        if (cantidad < MAX) {
            int i = cantidad;

            // Lógica de inserción ordenada:
            // Desplazamos a la derecha mientras la prioridad del ticket nuevo 
            // sea mayor que la de los que ya están en la cola.
            while (i > 0 && nivelesPrioridad[i - 1] < prioridad) {
                nivelesPrioridad[i] = nivelesPrioridad[i - 1];
                idsTickets[i] = idsTickets[i - 1];
                i--;
            }

            // El ticket del servidor (999) quedará en el índice 0, 
            // desplazando al fondo de pantalla (0) al final.
            idsTickets[i] = idTicket;
            nivelesPrioridad[i] = prioridad;
            cantidad++;
        }
    }

    @Override
    public void Desencolar() {
        if (!ColaVacia()) {
            // Estrategia 2: Extraemos el de mayor prioridad (servidor)
            // y todos los tickets de fondo de pantalla avanzan un lugar.
            for (int i = 0; i < cantidad - 1; i++) {
                idsTickets[i] = idsTickets[i + 1];
                nivelesPrioridad[i] = nivelesPrioridad[i + 1];
            }
            cantidad--;
        }
    }

    @Override
    public int Primero() {
        // Devuelve el ID del ticket con prioridad 999 si existe, sino el siguiente más alto
        return idsTickets[0];
    }

    @Override
    public int Prioridad() {
        return nivelesPrioridad[0];
    }

    @Override
    public boolean ColaVacia() {
        return cantidad == 0;
    }
}
